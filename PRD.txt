# Product Requirements Document: StayNest - Vacation Rental Marketplace

## 1. Project Overview

### Project Name
StayNest - Vacation Rental Marketplace

### Project Type
Vertical market platform for vacation rentals and short-term accommodations

### Core Functionality Summary
StayNest is a peer-to-peer marketplace connecting property owners with travelers seeking unique accommodation experiences. The platform enables listing management, search with map integration, booking workflows, review systems, and secure payments.

### Target Users
- Travelers seeking short-term accommodations (primary user base)
- Property owners and hosts listing their spaces
- Platform administrators managing content and users

## 2. User Roles and Authentication

### User Roles & Permissions

- Guest: Unauthenticated browsing user (Browse listings, view public content)
- User: Registered member seeking accommodations (Search, book, review, manage profile)
- Host: Property owner listing spaces (Create listings, manage bookings, set pricing)
- Admin: Platform administrator (Manage users, listings, disputes, analytics)

### SignUp Page

- Email/password input fields
- First name and last name fields
- Google OAuth button
- Apple Sign-In button
- Terms of service checkbox
- "Create Account" button
- Link to sign-in page

### SignUp Flow

1. User clicks "Sign Up" button
2. User enters email, password, first name, last name OR selects OAuth provider
3. System validates input (email format, password strength 8+ chars)
4. System checks if email already exists
5. If valid: System creates user record, generates JWT token
6. Token stored in localStorage
7. User redirected to onboarding or dashboard

### SignIn Page

- Email input field
- Password input field
- "Forgot Password" link
- Google OAuth button
- Apple Sign-In button
- Magic link option
- "Sign In" button
- Link to sign-up page

### SignIn Flow

1. User enters email and password OR uses OAuth/Magic link
2. System validates credentials
3. If valid: System generates JWT token with 24h expiry and refresh token with 7-day sliding window
4. Tokens stored in localStorage
5. User redirected to previous page or dashboard
6. If expired: System attempts token refresh
7. If refresh fails: User prompted to re-authenticate

### Session Management

- JWT tokens stored in localStorage with 24-hour expiry
- Refresh tokens with 7-day sliding window expiration
- Automatic logout on token expiration
- Session display in header showing user avatar and name

## 3. User Flows

### Search and Discovery Page

- Hero section with large background image and search bar
- Destination input with autocomplete
- Check-in and check-out date pickers
- Guest count selector (adults, children, infants)
- "Search" button
- Filter panel (collapsible on mobile)

### Search and Discovery Flow

1. User lands on homepage with search bar prominent
2. User enters destination (city, neighborhood, or zip code)
3. User selects check-in date (calendar widget, min: today)
4. User selects check-out date (calendar widget, max: 30 days ahead)
5. User selects guest count (adults: 1-16, children: 0-10, infants: 0-5)
6. User clicks "Search" button
7. System displays results page with map view (right side, 50% width on desktop) and list view (left side, scrollable card grid)
8. User applies filters: price range, property type, bedrooms, bathrooms, amenities, instant booking, superhost only
9. User adjusts map viewport to refine results
10. User clicks listing card to view details

### Listing Detail Page

- Image gallery with horizontal scroll and thumbnails (5+ photos)
- Fullscreen lightbox on image click
- Title, location, rating with review count
- Host avatar and name with Superhost badge
- Price per night, cleaning fee, service fee
- Description section with expandable text
- Amenities grid with icons
- Location map (interactive, zoomable)
- House rules and policies
- Cancellation policy
- Reviews section with sorting options
- Similar listings carousel
- Sticky booking widget (right sidebar on desktop, bottom fixed on mobile)

### Listing Detail Flow

1. User clicks listing card from search results
2. Page loads listing detail view (/listing/:id)
3. Image gallery displays with horizontal scroll and thumbnail navigation
4. User can click main image to open fullscreen lightbox
5. Key information displayed in listing header section
6. User scrolls to read description, view amenities, check location map
7. User reviews house rules, cancellation policy, and reviews section
8. User selects dates in booking widget
9. User clicks "Reserve" button
10. User redirected to checkout if not logged in, else continues to payment flow

### Booking Page

- Listing thumbnail and title
- Dates display (check-in, check-out, number of nights)
- Nightly rate breakdown
- Cleaning fee display
- Service fee display
- Taxes display
- Total amount
- Payment method selection (credit card form or saved method)
- Guest information inputs (adults, children, infants count)
- Special requests textarea
- House rules checkbox
- "Confirm and Pay" button

### Booking Flow

1. User clicks "Reserve" on listing detail
2. System checks authentication and redirects to login if needed
3. Booking summary page displays with price breakdown
4. User enters payment method via Stripe Elements or selects saved method
5. User enters guest count and optional special requests
6. User reviews and accepts house rules
7. User clicks "Confirm and Pay"
8. System processes payment via Stripe with payment intent creation, card tokenization, and charge authorization
9. On success: Booking created, confirmation page displays with confirmation code and check-in instructions
10. Confirmation email sent to both user and host

### Host Listing Creation Page

- Multi-step wizard interface (8 steps)
- Progress indicator showing current step
- Location step: Address autocomplete, property type selector, map marker verification
- Property details step: Bedrooms, bathrooms, max occupancy, square footage inputs
- Amenities step: Category checkboxes with preset amenities
- Photos step: Drag-and-drop upload area, reorderable photo gallery, cover photo selector
- Title and description step: Title input (100 char max), description textarea (100-2000 chars)
- Pricing step: Base nightly price, weekly/monthly discounts, cleaning fee, extra guest fee, smart pricing suggestion
- Calendar step: Availability calendar, blocked dates, minimum/maximum stay settings
- Review step: Full listing preview before publishing

### Host Listing Creation Flow

1. Host logs in and navigates to "Become a Host" from menu
2. Host clicks "Start your listing" to begin multi-step wizard
3. Step 1: Location - Enter property address with autocomplete, select property type, verify map marker position
4. Step 2: Property Details - Enter number of bedrooms, bathrooms, max occupancy, and optional square footage
5. Step 3: Amenities - Select from preset amenity categories and check included amenities
6. Step 4: Photos - Upload minimum 5 photos (min 1024x768, max 20MB each), arrange order, set cover photo
7. Step 5: Title and Description - Enter listing title (max 100 characters) and description (min 100, max 2000 characters)
8. Step 6: Pricing - Set base nightly price, optional weekly/monthly discounts, cleaning fee, extra guest fee
9. Step 7: Calendar - Set availability, block specific dates, configure minimum and maximum stay requirements
10. Step 8: Review - Preview listing as it will appear, make edits if needed, click "Publish Listing"
11. System publishes listing with "Pending Review" status and notifies admin
12. Host receives confirmation email
13. Listing goes live after admin content moderation approval

### Review Flow

1. After check-out date, system sends review request email to guest
2. User visits "Trips" page in dashboard
3. User sees completed trip with "Write a Review" button
4. User clicks to write review and enters overall rating (1-5 stars)
5. User enters review text (min 20 characters, max 5000 characters)
6. User rates specific categories: cleanliness, accuracy, check-in, communication, location, value (1-5 stars each)
7. User optionally uploads up to 10 photos (max 10MB each)
8. User clicks "Submit Review"
9. System publishes review (visible after both parties submit or after 14 days grace period)
10. Host receives notification and can respond publicly
11. Listing average rating updates based on new review

## 6. Error Handling

### Error Codes

- VALIDATION_ERROR (400): Invalid input data
- UNAUTHORIZED (401): Authentication required
- FORBIDDEN (403): Insufficient permissions
- NOT_FOUND (404): Resource not found
- CONFLICT (409): Resource conflict (e.g., email taken)
- PAYMENT_FAILED (402): Payment processing error
- RATE_LIMITED (429): Too many requests
- INTERNAL_ERROR (500): Server error

### User-Facing Error Messages

- Invalid login: "Incorrect email or password. Please try again."
- Email taken: "An account with this email already exists."
- Payment declined: "Your payment could not be processed. Please try a different payment method."
- Listing unavailable: "This listing is no longer available for the selected dates."
- Booking conflict: "These dates are no longer available. Please select different dates."
- Network error: "Connection lost. Please check your internet and try again."
- Session expired: "Your session has expired. Please sign in again."

### Empty States

- No search results: "No listings found for your search. Try adjusting your filters or dates." (Show reset filters button)
- No bookings: "You have no trips yet. Start exploring!" (Show search button)
- No reviews: "This listing has no reviews yet. Be the first to stay!"
- No saved listings: "You haven't saved any listings yet." (Show explore button)
- No messages: "No messages yet. Start a conversation with a host." (Show browse listings button)

### Loading States

- Search results: Skeleton cards (6 items) with pulsing animation
- Listing detail: Full-page skeleton with image carousel placeholder
- User profile: Avatar circle with text placeholder lines
- Booking form: Disabled form with spinner overlay on submit button

## 8. Admin Tools

### Admin Dashboard Features

- User management: Search, view, suspend, delete users
- Listing moderation: Review pending listings, approve/reject
- Content flagging: Review reported listings/reviews
- Booking management: View all bookings, process refunds
- Dispute resolution: Handle payment disputes, guest/host conflicts
- Analytics: Platform metrics, revenue reports, user growth
- Pricing oversight: Review pricing anomalies, discount compliance

### Moderation Workflow

```
1. Host submits listing for review
2. Admin receives notification with listing details
3. Admin reviews: Photos, Description, Pricing, Amenities
4. Admin approves or rejects with reason
5. Host notified of decision via email
6. If rejected, host can edit and resubmit
```

## 10. Performance Requirements

### Performance Techniques

- Image optimization: WebP format, responsive srcset, lazy loading
- Code splitting: Route-based splitting with React.lazy
- Caching: SWR for API caching, 5-minute stale time
- CDN: Static assets served from CDN edge
- Compression: Gzip/Brotli for text assets
- Prefetching: Prefetch search results on focus
- Database indexes: Composite indexes on search fields
- Query optimization: Pagination, cursor-based for large datasets

### Asset Budgets

- Initial JS bundle: 200KB gzipped (Build-time check)
- Initial CSS: 50KB gzipped (Build-time check)
- Largest image (LCP): 150KB (Build-time check)
- Total page weight: 3MB (Performance budget)

## 11. Accessibility

### WCAG 2.1 AA Compliance

| Requirement | Implementation |
|-------------|----------------|
| Color contrast | 4.5:1 for text, 3:1 for UI components |
| Keyboard navigation | Full site navigable via keyboard |
| Focus indicators | Visible focus rings on all interactive elements |
| Screen reader | Semantic HTML, ARIA labels for complex widgets |
| Form labels | All inputs have associated labels |
| Error identification | Errors announced and linked to fields |

### Reduced Motion `prefers-reduced-motion`

- Page transitions: Slide + fade (300ms) on standard | Instant on reduced motion
- Scroll reveals: Fade up (400ms ease-out) on standard | None on reduced motion
- Image galleries: Slide (250ms) on standard | Instant swap on reduced motion
- Micro-interactions: Scale on hover (150ms) on standard | None on reduced motion
- Loading spinners: Rotate (1s infinite) on standard | Static indicator on reduced motion

### Accessibility Features

- Skip navigation link at page top
- Breadcrumb navigation for listing pages
- Landmark regions (header, main, footer, nav)
- Heading hierarchy (h1 for page title, h2 for sections)
- Alt text for all images
- Focus trap in modal dialogs
- Escape key closes modals
